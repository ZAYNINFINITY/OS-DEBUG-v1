Tutorial / Walkthrough

Workflow: Briefing -> Evidence Collection -> Diagnosis -> Mitigation -> Outcome.

Simulator commands: help, status, tools, objectives, hint, clear.

Evidence commands: case-specific probes such as inspect, logs, trace, strace, tree, map, probe.

Diagnosis/Fix format: diagnose <root-cause> and fix <mitigation>.

Timer and stability: incorrect interventions reduce stability; timeout triggers failure.

Stats: each run is logged for comparative learning and strategy review.

Real terminal context (outside simulator):

pwd -> show current directory.

cd /srv/main-server -> move to service folder.

ls -> list files/log bundles/tools.

cat incident.log -> inspect raw log snippets quickly.
